# writeup/

Analysis documents, vulnerability reports, and platform submissions.

## Contents

| File | Purpose |
|------|---------|
| `submission.md` | **Primary** — Platform submission draft (auto-generated by `/new-finding`) |
| `analysis.md` | Optional — Deep technical analysis, root cause investigation |
| `patch-diff.md` | Optional — If you identify the vulnerable code and its fix |

## Agent Instructions

- `submission.md` is the canonical report — populate it before running `/submit`
- Use `/submit <finding-name>` to auto-fill `submission.md` from `finding.md` data
- Reference evidence files using relative paths: `../evidences/YYYY-MM-DD_NN_description.png`
- Reference PoC files using relative paths: `../poc/exploit.py`
- Steps to Reproduce in `submission.md` must be literal and copy-paste ready — a triager must be able to reproduce with zero prior context
- Do not submit to the platform directly — have a human review `submission.md` first
